# 05) 비기능 요구사항 + 디자인 시스템(v1)

최종 업데이트: 2026-04-02

## 9) 비기능 요구사항

### 9.1 접근성
- 버튼 텍스트 명확(“문의하기”만 남발하지 말고 채널 명시)
- 대비(텍스트/배경), 터치 영역(최소 44px)
- 폼 에러는 색만으로 구분하지 않기(텍스트 안내 포함)

### 9.2 성능
- 이미지 최적화(웹용 리사이즈, lazy-load)
- 폰트 최소화(필요 시 system font 우선)
- 불필요한 애니메이션 제한(특히 모바일)

### 9.3 안정성
- 외부 링크(카카오/지도/SNS) 깨짐 대비: 링크 유효성 점검 체크리스트 포함

## 9.4 디자인 시스템 (v1: 블랙 베이스 + 골드 포인트)

> 목표 톤: “럭셔리/프리미엄/단정한 골프 스튜디오”.  
> 구현 원칙: **CSS 변수(토큰)로 색/라운드/그림자/타이포를 추상화**해서, 추후 “골드 ↔ 그린” 같은 테마 변경이 `토큰만` 바꾸면 되도록 한다.

### 9.4.1 브랜드/법적 메모(주의)
- “구찌 느낌”은 **무드(블랙+골드, 세리프 헤딩, 미니멀 레이아웃)**로만 차용하고, 특정 브랜드의 **로고/패턴/문구/상표를 직접 사용하지 않는다.**

### 9.4.2 컬러 토큰(권장값)
- Background
  - `--bg`: `#070708` (페이지 기본)
  - `--surface`: `#0E0E10` (카드/섹션)
  - `--surface-2`: `#141418` (인풋/강조 카드)
- Text
  - `--text`: `#F5F5F6` (본문)
  - `--muted`: `#B3B3B8` (보조 텍스트)
  - `--border`: `rgba(255,255,255,0.10)` (구분선)
- Accent (Gold)
  - `--accent`: `#C8A24A` (메인 포인트)
  - `--accent-2`: `#E2C77A` (hover/하이라이트)
  - `--accent-ink`: `#0B0B0C` (골드 배경 위 텍스트)
- State
  - `--danger`: `#FF5C5C`
  - `--success`: `#35D07F`

### 9.4.3 타이포(권장)
- 헤딩(세리프, 럭셔리 톤):
  - 폰트 후보: `Playfair Display`(영문), `Noto Serif KR`(국문) 또는 `Noto Serif KR` 단독
  - 사용 규칙: H1/H2만 세리프, 본문은 산세리프로 가독성 확보
- 본문(산세리프, 실용):
  - 폰트 후보: `Pretendard`(권장) 또는 시스템 폰트 스택
- 기본 사이즈(권장):
  - Body: 16px / line-height 1.6
  - H1: 36–44px(모바일 28–34px), letter-spacing 약간(-0.02em)

### 9.4.4 레이아웃/컴포넌트 기본 규칙
- 섹션 폭: max-width 1100~1200px, 좌우 padding 16~20px(모바일)
- 라운드: `--radius` 14px(카드), 버튼은 12px
- 그림자: 과하지 않게(다크 배경에서는 “테두리 + 아주 약한 그림자” 위주)
- 카드: `surface` 배경 + 1px border + 내부 여백 20~24px

### 9.4.5 버튼(필수: CTA 3종 + 상태)
- Primary(문의 폼 / 대표 CTA):
  - 배경: `--accent`, 텍스트: `--accent-ink`
  - Hover: `--accent-2`
- Secondary(카카오 상담):
  - 기본: 투명 + 테두리 `--accent` + 텍스트 `--accent`
  - Hover: 배경 `rgba(200,162,74,0.12)`
- Tertiary(전화):
  - 기본: `--surface-2` + border `--border` + 텍스트 `--text`
  - Hover: border 조금 더 진하게 + 배경 살짝 밝게
- Disabled:
  - opacity 0.5, cursor not-allowed
- 로딩:
  - 버튼 높이 고정(레이아웃 흔들림 방지), 스피너는 `--accent-ink`/`--text` 대비 유지

### 9.4.6 폼(UI)
- 인풋 배경: `--surface-2`, 텍스트 `--text`, placeholder `--muted`
- 포커스 링: `--accent` 2px(또는 box-shadow)
- 에러: `--danger` + 에러 메시지 텍스트(색만 쓰지 않음)

### 9.4.7 CSS 변수 예시(구현 참고)
```css
:root{
  --bg:#070708; --surface:#0E0E10; --surface-2:#141418;
  --text:#F5F5F6; --muted:#B3B3B8; --border:rgba(255,255,255,.10);
  --accent:#C8A24A; --accent-2:#E2C77A; --accent-ink:#0B0B0C;
  --danger:#FF5C5C; --success:#35D07F;
  --radius:14px;
}
```

### 9.4.8 테마 교체(골드 ↔ 그린) 정책
- 구현 시 **색상 직접값(HEX)을 컴포넌트에 박지 않는다.** 전부 `--accent`/`--bg` 같은 토큰만 사용한다.
- “그린 테마”는 별도 토큰 세트로 제공(예: `data-theme="green"`에서 `--accent`만 교체)하도록 설계한다.
