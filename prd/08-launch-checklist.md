최종 업데이트: 2026-04-02

## PHASE 1 — 콘텐츠 (배포 3일 전)

### 플레이스홀더 전체 교체
- [ ] `{{PHONE_NUMBER}}` → `010-8441-3065` 교체 확인
- [ ] `{{KAKAO_URL}}` → 실제 카카오 채널 URL 교체
- [ ] `{{COACH_NAME}}` × 4명 전부 입력
- [ ] `{{COACH_BIO_SHORT}}` × 4명 전부 입력
- [ ] `{{COACH_CREDENTIALS}}` × 4명 전부 입력
- [ ] `{{PROGRAM_FIELD_DESC}}` 입력
- [ ] `{{PROGRAM_SHORTGAME_DESC}}` 입력
- [ ] `{{PROGRAM_OUTSIDE_DESC}}` 입력
- [ ] `{{PRICING_POLICY_SHORT}}` 입력
- [ ] `{{REFUND_POLICY_SHORT}}` 입력
- [ ] `{{BUSINESS_HOURS}}` 입력
- [ ] `{{PARKING_INFO}}` (주차 시간/방식 확인 후) 입력
- [ ] `{{BUSINESS_INFO}}` (푸터 사업자 정보) 입력
- [ ] `{{LEAD_NOTIFY_EMAIL}}` Gmail 주소 입력

### 링크 유효성
- [ ] 전화번호 `tel:010-8441-3065` 클릭 시 실제 전화 연결 확인
- [ ] 카카오 채널 링크 클릭 시 실제 채널 열리는지 확인
- [ ] 구글맵 링크 정상 작동 확인
- [ ] 네이버 플레이스 링크 정상 작동 확인
- [ ] 유튜브 채널 링크 확인 (`@good_ggolf`)
- [ ] 인스타 링크 확인 (`good_ggolf`)
- [ ] 개인정보처리방침 링크 작동 확인

## PHASE 2 — 기능 (배포 2일 전)

- [ ] 연락처 필드 — 잘못된 번호 형식 시 에러 메시지 표시
- [ ] 개인정보 동의 체크 안 하면 제출 불가 확인
- [ ] 허니팟 필드(`company`) 숨겨져 있는지 확인

- [ ] 정상 제출 시 → Thank-you 화면 표시 확인
- [ ] 제출 실패 시 → 입력값 유지, 에러 메시지 표시 확인

- [ ] 하단 CTA 섹션 작동 확인
- [ ] 모바일 Sticky CTA 3개 작동 확인

## PHASE 3 — 디자인/UX (배포 2일 전)

### 모바일 확인 (최우선)
- [ ] iPhone Safari 실기기 확인
- [ ] Android Chrome 실기기 확인
- [ ] Sticky CTA가 콘텐츠를 가리지 않는지 확인
- [ ] 터치 타깃 최소 44px 확인 (버튼이 너무 작지 않은지)
- [ ] 폼 입력 시 키보드 올라올 때 레이아웃 깨지지 않는지 확인

### 데스크탑 확인
- [ ] Chrome 최신 버전
- [ ] Safari 최신 버전
- [ ] Edge 최신 버전

### 전체 스크롤 확인
- [ ] 모든 섹션 순서대로 스크롤하며 깨진 레이아웃 없는지 확인
- [ ] 이미지 전부 로드되는지 확인 (깨진 이미지 없음)
- [ ] 폰트 정상 로드 확인 (FOUT/FOIT 없음)

## PHASE 5 — SEO/메타 (배포 1일 전)

- [ ] `<title>` 태그 확인
- [ ] `<meta description>` 확인
- [ ] OG 태그 확인 (카카오톡으로 링크 공유해서 썸네일 미리보기 확인)
- [ ] Schema.org LocalBusiness JSON-LD 삽입 확인
- [ ] `sitemap.xml` 생성 및 접근 가능 확인 (`/sitemap.xml`)
- [ ] `robots.txt` 생성 확인 (`/robots.txt`)
- [ ] favicon 등록 확인 (브라우저 탭에 아이콘 표시)

## PHASE 8 — 배포 직전 (당일)

- [ ] 환경변수 전부 프로덕션 값으로 설정 (테스트용 키 없는지 확인)
- [ ] 구글 시트 접근 권한 확인 (서비스 계정 or 공유 설정)
- [ ] 도메인 연결 확인 (DNS 전파 완료)
- [ ] HTTPS 인증서 정상 확인 (자물쇠 아이콘)
- [ ] 404 페이지 확인 (없으면 기본 페이지라도 있어야 함)

## PHASE 9 — 배포 후 (당일~3일)

- [ ] 구글 서치 콘솔에 사이트맵 제출
- [ ] 네이버 서치어드바이저에 사이트 등록 및 사이트맵 제출
- [ ] 구글맵 플레이스 URL 업데이트 (랜딩 URL로)
- [ ] 네이버 플레이스 URL 업데이트
- [ ] 인스타 바이오 링크 → 랜딩 URL로 변경
- [ ] 유튜브 채널 링크 → 랜딩 URL로 변경
- [ ] 테스트 폼 제출로 전체 운영 파이프라인 최종 확인
- [ ] 클라이언트(동생)에게 최종 URL 전달 및 확인 요청

## 사고 발생 시 대응 프로토콜

| 상황 | 즉시 행동 |
|------|----------|
| 폼이 제출 안 됨 | 전화/카카오 CTA는 살아있음 → 클라이언트에 즉시 보고, 원인 파악 |
| 외부 링크 깨짐 | 해당 버튼 임시 숨김 처리 후 수정 배포 |
| 페이지 오류(500 등) | Vercel 로그 확인 → 이전 배포로 롤백 |
| 개인정보 관련 민원 | 즉시 클라이언트에 에스컬레이션, 수집 데이터 파기 절차 진행 |
